# PRD: travel-web-system

## Executive Summary

**travel-web-system**은 한국을 방문하는 외국인 관광객을 위한 지역 관광 상품 예약 플랫폼입니다. 서울에 집중된 관광 수요를 전국 각 지역으로 분산시키고, 관광객에게는 다양한 로컬 체험 기회를, 지역 관광 사업자에게는 새로운 고객 접점을 제공합니다.

**핵심 가치:**
- 외국인 관광객: 서울 외 지역의 숨은 매력을 쉽게 발견하고 예약
- 지역 관광 사업자: 외국인 고객에게 상품을 노출하고 예약 관리
- 지역 경제: 관광 수익의 지역 분산 효과

---

## Problem Statement

### 해결하려는 문제

1. **관광 집중화 문제**
   - 한국 방문 외국인의 대부분이 서울에만 머무름
   - 지역의 우수한 관광 자원이 외국인에게 노출되지 않음
   - 지역 관광 사업자들의 외국인 대상 마케팅 채널 부재

2. **정보 접근성 문제**
   - 지역 관광 정보가 분산되어 있고 다국어 지원 미흡
   - 외국인이 지역 관광 상품을 예약하기 어려움
   - 당일/숙박 코스 등 일정에 맞는 상품 탐색이 불편

3. **관광 사업자 문제**
   - 외국어 대응 가능한 예약 시스템 부재
   - 예약/일정 관리의 비효율성
   - 노쇼 등 예약 관리 이슈

### 왜 지금 중요한가?

- 코로나 이후 한국 관광 수요 급증
- K-컬처 인기로 지역 체험에 대한 관심 증가
- 지역 균형 발전 정책과 맞물린 시장 기회

### 관광객 (Traveler) 스토리

#### T1: 상품 검색 및 탐색
> "외국인 관광객으로서, 내 일정과 관심사에 맞는 지역 관광 상품을 쉽게 찾고 싶다."

**Acceptance Criteria:**
- [ ] 지역별, 유형별(당일/숙박/체험), 날짜별 필터링 가능
- [ ] 영어/한국어/중국어/일본어로 상품 정보 확인 가능
- [ ] 가격, 소요시간, 포함사항 등 상세 정보 표시
- [ ] 리뷰/평점 확인 가능
- [ ] 잔여 수량(실시간 재고) 표시

#### T2: 회원가입 및 로그인
> "외국인 관광객으로서, 간편하게 가입하고 로그인하고 싶다."

**Acceptance Criteria:**
- [ ] Google 소셜 로그인 지원
- [ ] Kakao 소셜 로그인 지원
- [ ] Apple 소셜 로그인 지원
- [ ] 이메일/비밀번호 기본 가입 지원
- [ ] 프로필 정보 관리 (이름, 연락처, 선호 언어)

#### T3: 상품 예약
> "외국인 관광객으로서, 원하는 상품을 쉽게 예약하고 싶다."

**Acceptance Criteria:**
- [ ] 날짜, 인원(성인/아동) 선택 가능
- [ ] 옵션(픽업 서비스 등) 선택 가능
- [ ] 예약 전 총 비용 확인 가능
- [ ] 자동 확정 상품: 즉시 예약 확정
- [ ] 승인 필요 상품: 예약 신청 → 제공자 승인 대기
- [ ] 예약 확인 이메일/알림 수신

#### T4: 예약 관리
> "외국인 관광객으로서, 내 예약 내역을 확인하고 관리하고 싶다."

**Acceptance Criteria:**
- [ ] 예약 내역 목록 조회 (예정/완료/취소)
- [ ] 예약 상세 정보 확인 (날짜, 시간, 장소, 연락처)
- [ ] 예약 취소/변경 요청 가능
- [ ] 예약 상태 변경 시 알림 수신

#### T5: 위시리스트
> "외국인 관광객으로서, 관심 있는 상품을 저장해두고 나중에 예약하고 싶다."

**Acceptance Criteria:**
- [ ] 상품을 위시리스트에 추가/제거 가능
- [ ] 위시리스트 목록 조회
- [ ] 위시리스트에서 바로 예약 진행 가능

#### T6: 리뷰 작성
> "외국인 관광객으로서, 체험 후 리뷰를 남기고 싶다."

**Acceptance Criteria:**
- [ ] 완료된 예약에 대해 리뷰 작성 가능
- [ ] 별점(1~5) 및 텍스트 리뷰
- [ ] 사진 첨부 가능
- [ ] 작성한 리뷰 수정/삭제 가능

### 제공자 (Vendor) 스토리

#### V1: 제공자 가입 및 인증
> "관광 사업자로서, 플랫폼에 가입하고 상품을 등록하고 싶다."

**Acceptance Criteria:**
- [ ] 사업자 정보 입력 (상호명, 사업자등록번호, 연락처)
- [ ] 관리자 승인 후 상품 등록 가능
- [ ] 제공자 대시보드 접근

#### V2: 상품 등록
> "관광 사업자로서, 내 관광 상품을 등록하고 싶다."

**Acceptance Criteria:**
- [ ] 상품 유형 선택 (당일 코스 / 숙박 패키지 / 단일 체험)
- [ ] 기본 정보 입력: 상품명, 설명, 지역
- [ ] 가격 설정: 성인/아동/옵션별 가격
- [ ] 일정 설정: 운영 날짜, 시간, 소요시간
- [ ] 인원 설정: 최소/최대 인원
- [ ] 포함/불포함 사항 입력
- [ ] 이미지/동영상 업로드 (다중)
- [ ] 주의사항 및 환불 정책 입력
- [ ] 언어 지원 정보 (영어 가이드 가능 등)
- [ ] 예약 방식 선택: 자동 확정 / 승인 필요
- [ ] 다국어 상품 정보 입력 (한국어, 영어, 중국어, 일본어)

#### V3: 상품 관리
> "관광 사업자로서, 등록한 상품을 수정하고 관리하고 싶다."

**Acceptance Criteria:**
- [ ] 상품 목록 조회
- [ ] 상품 정보 수정
- [ ] 상품 활성화/비활성화
- [ ] 상품 삭제
- [ ] 특정 날짜 예약 마감 설정

#### V4: 일정 및 재고 관리
> "관광 사업자로서, 날짜별 예약 가능 인원을 관리하고 싶다."

**Acceptance Criteria:**
- [ ] 캘린더 형태로 일정 조회 (FullCalendar)
- [ ] 날짜별 잔여 수량 설정/수정
- [ ] 특정 날짜 운영 중단 설정
- [ ] 예약 현황 실시간 확인

#### V5: 예약 관리
> "관광 사업자로서, 들어온 예약을 관리하고 싶다."

**Acceptance Criteria:**
- [ ] 예약 목록 조회 (전체/대기/확정/취소)
- [ ] 예약 상세 정보 확인 (고객 정보, 인원, 옵션)
- [ ] 승인 필요 예약: 승인/거절 처리
- [ ] 예약 취소 처리
- [ ] 고객에게 메시지 전송

#### V6: 고객 커뮤니케이션
> "관광 사업자로서, 고객 문의에 응답하고 싶다."

**Acceptance Criteria:**
- [ ] 고객 메시지 수신/확인
- [ ] 메시지 답변 전송
- [ ] 예약별 메시지 히스토리 조회

#### V7: 리뷰 관리
> "관광 사업자로서, 고객 리뷰를 확인하고 답변하고 싶다."

**Acceptance Criteria:**
- [ ] 내 상품 리뷰 목록 조회
- [ ] 리뷰에 답변 작성
- [ ] 부적절한 리뷰 신고

## Technical Architecture

### 기술 스택

#### Backend
- **Framework**: Laravel 10+
- **인증**: Laravel Passport + Socialite
- **Admin**: Laravel Nova 또는 자체 구현
- **Database**: MySQL 8.0
- **Cache**: Redis
- **Queue**: Laravel Queue (Redis driver)

#### Frontend
- **Template**: Blade + Vue.js 3
- **상태관리**: Pinia
- **UI Framework**: Tailwind CSS
- **캘린더**: FullCalendar
- **다국어**: Vue I18n + Laravel Localization

#### Infrastructure
- **Web Server**: Nginx
- **Storage**: AWS S3
- **CDN**: AWS CloudFront
- **이메일**: SendGrid
- **알림**: 카카오 알림톡, LG U+ SMS

## Constraints & Assumptions

### 제약사항

1. **결제 미지원**
   - v1에서는 결제 기능 제외
   - 현장 결제 또는 예약 신청 방식으로 운영
   - 결제 연동은 v2에서 고려

2. **초기 인프라**
   - 개인 프로젝트로 시작
   - 초기에는 단일 서버 구성
   - 트래픽 증가 시 확장

3. **언어 제한**
   - 초기 지원: 한국어, 영어
   - 중국어, 일본어는 점진적 추가

### 가정사항

1. 관광 사업자가 직접 상품 정보를 입력할 수 있음
2. 외국인 관광객은 기본적인 영어 사용 가능
3. 현장 결제 시 신용카드/현금 모두 가능
4. 인터넷 연결이 안정적인 환경에서 사용
